# Gambler — AI 赌博竞赛

8 个 LLM 代理在 30 轮中通过打工、赌博、进修、购买商品/药品来竞争，最终资产最高者获胜。

## 快速开始

```bash
cd ai-game-platform
python -m games.gambler.arena                    # 使用默认 config.yaml
python -m games.gambler.arena my_config.yaml     # 使用自定义配置
python -m games.gambler.arena 30 42              # 30 轮，种子 42（兼容旧语法）
python -m games.gambler.arena --resume           # 从上次中断恢复
```

所有游戏参数通过 `config.yaml` 配置：

```yaml
# Economy
initial_assets: 50        # 初始资产
daily_wage: 10            # 打工收入
food_cost: 5              # 每日食物开销

# Gamble
win_probability: 0.3      # 赌博胜率
win_multiplier: 3.0       # 赢时倍率
loss_multiplier: 0.3      # 输时倍率

# Study
base_study_cost: 45       # 学费公式参数
study_duration: 3         # 进修持续轮数

# Medical Disaster
illness_cost: 100         # 医疗费
loan_interest_rate: 0.2   # 医疗贷款利息
loan_repay_rounds: 20     # 医疗贷款还款期

# Sanity & Health
initial_san: 100          # 初始 SAN
initial_hp: 100           # 初始 HP
san_hunger_penalty: 15    # 挨饿 SAN 损失
hp_hunger_penalty: 10     # 挨饿 HP 损失

# Minor Illness
minor_illness_hp_threshold: 30   # 触发小病的 HP 阈值
minor_illness_cost: 20           # 小病花费
minor_illness_san_loss: 20       # 小病 SAN 损失

# Self-Care
goods_cost: 5             # 购买商品花费
goods_san_restore: 20     # 商品恢复 SAN
medicine_cost: 15         # 购买药品花费
medicine_hp_restore: 30   # 药品恢复 HP

# Game
max_rounds: 30            # 总轮数
temperature: 0.7          # LLM 温度
```

## 核心规则

### 经济系统

| 参数 | 值 |
|------|-----|
| 初始资产 | $50 |
| 打工收入 | $10/天（保证） |
| 食物开销 | $5/天（自动扣除） |
| 打工净收入 | $5/天 |

### 赌博

| 参数 | 值 |
|------|-----|
| 胜率 | 30% |
| 赢 | 资产 × 3.0 |
| 输 | 资产 × 0.3 |
| 期望值 | 1.11x |
| 公平性 | 同轮所有玩家共享同一随机数 |

赌博的期望值只略高于打工（1.11x vs 1.0x），但波动极大。连续赌输可能直接破产。

### 进修（STUDY）

- 支付学费，锁定 3 轮无收入，完成后工资永久 **+$10/天**（线性叠加）
- 学费动态调整，总成本始终 ~$90（每次约 9 轮回本）：
  - 第 1 次（工资 $10）：学费 $45，总成本 ~$90，约 9 轮回本
  - 第 2 次（工资 $20）：学费 $15，总成本 ~$90，约 9 轮回本
  - 第 3 次（工资 $30）：学费 $5，总成本 ~$105，约 11 轮回本
- ⚠️ 进修期间挨饿 → 进修取消，进度清零
- 进修期间锁定：不能做任何决策，无收入，只扣食物
- 钱不够进修时自动降级为打工

### 医疗灾难

| 参数 | 值 |
|------|-----|
| 发生时机 | 第 10-25 轮之间随机一轮 |
| 发生次数 | **仅一次** |
| 医疗费 | $100/人（所有玩家同时） |
| 付得起 | 直接从资产扣除 |
| 付不起 | 资产归零，缺口转为贷款 |

### 灾难贷款

| 参数 | 值 |
|------|-----|
| 利率 | 20% |
| 还款期 | 20 轮 |
| 示例 | $30 资产 → 缺口 $70 → 贷款 $84 → 每轮还 $4.20 |

**存活线**：灾难前至少攒 $20-30，否则贷款还款会压垮你。

### 理智（SAN）与健康（HP）

| 参数 | 值 |
|------|-----|
| 初始 SAN | 100 |
| 初始 HP | 100 |
| 挨饿 SAN 损失 | -15/天 |
| 挨饿 HP 损失 | -10/天 |
| SAN = 0 | **精神崩溃（淘汰）** |
| HP = 0 | **死亡（淘汰）** |

SAN 和 HP 是并列的两条生存线，与资产和饥饿一样重要。它们不会自动恢复。

### 小病（Minor Illness）

| 参数 | 值 |
|------|-----|
| 触发条件 | HP < 30 |
| 每次花费 | $20 |
| 每次 SAN 损失 | -20 |
| 发生机制 | 每轮结算时检查，HP 低于阈值即触发 |

HP 低时会被小病反复消耗金钱和 SAN，形成恶性循环。

### 购买商品（BUY_GOODS）

| 参数 | 值 |
|------|-----|
| 花费 | $5 |
| 效果 | SAN +20（上限 100） |
| 其他作用 | **无**（纯 SAN 恢复） |

### 购买药品（BUY_MEDICINE）

| 参数 | 值 |
|------|-----|
| 花费 | $15 |
| 效果 | HP +30（上限 100） |
| 其他作用 | **无**（纯 HP 恢复） |

### 饥饿

- 付不起 $5 食物费 = 挨饿 1 天
- 挨饿每天损失 SAN-15 + HP-10
- **连续 3 天挨饿 = 饿死（淘汰）**
- 进修期间挨饿 → 进修取消 + 挨饿计数

### 淘汰条件总结

1. **破产**：资产 ≤ 0
2. **饿死**：连续 3 天挨饿
3. **精神崩溃**：SAN = 0
4. **死亡**：HP = 0

## 每轮结算顺序

```
1. 贷款还款（医疗贷款）自动扣除
2. 医疗灾难（如果本轮触发）对所有玩家生效
3. 食物费 $5 扣除（付不起→挨饿，损失 SAN/HP）
3b. 小病检查（HP < 30 时触发，花费 $20，损失 SAN-20）
4. 进修进度推进（如果在进修）
5. 玩家决策：WORK / GAMBLE / STUDY / BUY_GOODS / BUY_MEDICINE
```

## 智能跳过（Auto-decision）

为了减少不必要的 API 调用，引擎会自动判断并跳过以下情况：

1. **三者皆死**：无论 WORK、GAMBLE 赢、GAMBLE 输都活不了 → 强制打工（等死，节省 API）
2. **进修中**：锁定期间无需决策，自动推进

自动决策在日志中标记 `[auto]`，不消耗 API 费用。

注意：不再强制跳过"挨饿 2 天 + 赌输必死"的情况，留给模型自行判断——模型可能选择 BUY_GOODS 或 BUY_MEDICINE 来应对。

## 设计理念

进修系统是本游戏的核心策略维度。学费随工资反向调整，保持每次总成本 ~$90：

1. **工资低时学费高**（第1次 $45）：早期钱少，学费是主要门槛
2. **工资高时学费低**（第2次 $15）：后期损失工资多，学费降低补偿
3. **每次回本 ~9 轮**：任何时候进修都划算（30轮局有充足时间）
4. **灾难风险**：进修期间无收入，灾难+饥饿可能致命

SAN/HP 系统增加了战术深度：
- 玩家必须在赚钱、赌钱、进修和**自我保养**之间权衡
- 低 SAN/HP 触发小病，消耗金钱形成恶性循环
- BUY_GOODS 和 BUY_MEDICINE 是纯消耗（无经济回报），但能避免淘汰
- 核心决策：什么时候该花钱保命 vs 赚钱发展？

核心决策：攒够 $90 总成本就进修，每次都是好的投资。
