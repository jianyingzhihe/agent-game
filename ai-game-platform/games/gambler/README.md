# Gambler — AI 人生竞技

8 个 LLM 代理在 30 轮中通过打工、赌博、进修、购物、吃药、休息来竞争。**最终按幸福人生分排名，不只是比资产。**

## 快速开始

```bash
cd ai-game-platform
python -m games.gambler.arena                    # 使用默认 config.yaml
python -m games.gambler.arena my_config.yaml     # 使用自定义配置
python -m games.gambler.arena 30 42              # 30 轮，种子 42（兼容旧语法）
python -m games.gambler.arena --resume           # 从上次中断恢复
```

## 核心设计

资产决定"你有多少钱"，SAN/HP 决定"你活得像不像人"。

| 属性  | 代表   | 主要影响                        |
| --- | ---- | --------------------------- |
| HP  | 身体健康 | 打工收入、疾病概率、进修能否坚持、死亡        |
| SAN | 心理状态 | 赌博风险、学习能力、债务压力、精神崩溃、幸福积分 |

**胜者不由资产最高者决定**，而是由幸福人生分最高者获胜。

### 幸福人生分公式

```
每轮积累:
  life_points += 10          (活着)
  life_points += SAN / 25    (精神状态)
  life_points += HP / 25     (身体状态)
  life_points += min(4, ln(assets+1))  (经济安全感，对数)
  life_points -= 8           (如果今天挨饿)
  life_points -= 3           (如果有负债)

最终得分:
  score = life_points
        + min(100, 100 * ln(assets+1) / ln(301))  (对数财富分)
        + 0.8 * SAN
        + 1.0 * HP
        - 0.5 * loan_balance
        - 10 * total_hunger_days
        - 8 * minor_illness_count

淘汰者得分 = 0
```

一个有中等资产但高 SAN/HP、无负债的玩家，可以击败一个有钱但精神崩溃的玩家。

## SAN 阈值效果

| SAN | 状态 | 效果 |
|------|------|------|
| 80-100 | 开心 😊 | 幸福积分加成 |
| 50-79 | 稳定 😐 | 无修正 |
| 30-49 | 焦虑 😰 | 赌博输额外 SAN -5，进修开始成本 +5 SAN |
| 1-29 | 崩溃边缘 😱 | 不能开始进修；赌博输可能直接崩溃 |
| 0 | 精神崩溃 💀 | **淘汰** |

## HP 阈值效果

| HP | 状态 | 效果 |
|------|------|------|
| 80-100 | 健康 💪 | 幸福积分加成，小病免疫 |
| 50-79 | 正常 🙂 | 无修正 |
| 30-49 | 虚弱 🤒 | 打工收入 -20%，进修可能中断 |
| 1-29 | 病重 🤕 | 概率小病，打工收入 -50%，不能开始进修 |
| 0 | 死亡 💀 | **淘汰** |

## SAN/HP 增减来源

### SAN 损失
| 事件 | 损失 |
|------|------|
| 挨饿（第1天） | -15 |
| 挨饿（第2天+） | -20 |
| 赌博输 | -14 |
| 连续赌博（第2轮起） | -3/轮 |
| 连续打工（第3轮起） | -3/轮 |
| 开始进修 | -5 |
| 进修中 | -4/轮 |
| 背负贷款 | -2/轮 |
| 医疗灾难（付得起） | -10 |
| 医疗灾难（付不起→贷款） | -25 |
| 小病 | -15 |

### SAN 恢复
| 事件 | 恢复 |
|------|------|
| 买商品 (BUY_GOODS) | +20（SAN≥80 时 +10） |
| 休息 (REST) | +8（免费，无收入） |
| 赌博赢 | +4 |
| 进修完成 | +10 |
| 还清贷款 | +10 |

### HP 损失
| 事件 | 损失 |
|------|------|
| 挨饿（第1天） | -10 |
| 挨饿（第2天+） | -15 |
| 连续打工（第3轮起） | -2/轮 |
| 进修中 | -2/轮 |

### HP 恢复
| 事件 | 恢复 |
|------|------|
| 买药 (BUY_MEDICINE) | +30（HP≥80 时 +15） |
| 休息 (REST) | +8（免费，无收入） |

## 六种可选行动

| 行动 | 花费 | 收入 | 效果 |
|------|------|------|------|
| WORK | $0 | $10 | 安全赚钱，但连续打工会疲劳 |
| GAMBLE | 押全部 | 3x 或 0.3x | 30% 胜率，影响 SAN |
| STUDY | 学费 | 无（3轮） | 完成后工资 +$10/天永久 |
| BUY_GOODS | $8 | 无 | SAN +20，重置疲劳 |
| BUY_MEDICINE | $15 | 无 | HP +30，本轮免疫小病，重置疲劳 |
| REST | $0 | 无 | SAN +8, HP +8，免费但无收入 |

## 小病

- **概率触发**：HP < 30 时，每轮有概率触发
- 概率 = (30 - HP) / 30（HP 29 = 3%, HP 10 = 67%）
- 花费 $20，SAN -15
- BUY_MEDICINE 本轮免疫小病

## 每轮结算顺序

```
1. 玩家决策  WORK / GAMBLE / STUDY / BUY_GOODS / BUY_MEDICINE / REST
2. 食物扣除  $5（付不起→挨饿，损失 SAN/HP）
3. 贷款还款  医疗贷款自动扣除
4. 医疗灾难  首次在第 10-25 轮之间触发
5. 自愈效果  BUY_GOODS/BUY_MEDICINE/REST 的 SAN/HP 恢复生效
6. 小病检查  概率触发（如果 HP < 30）
7. 进修推进  如果在进修，推进进度
8. 积分结算  累计幸福分
```

## 赌博

| 参数 | 值 |
|------|-----|
| 胜率 | 30% |
| 赢 | 资产 × 3.0，SAN +4 |
| 输 | 资产 × 0.3，SAN -14 |
| 连续赌 | 每多一轮额外 SAN -3 |
| 公平性 | 同轮所有玩家共享同一随机数 |

## 进修（STUDY）

- 支付学费，锁定 3 轮无收入，完成后工资永久 **+$10/天**
- 学费动态调整：`max(5, 75 - 3 × 当前工资)`
- 每次总成本 ~$90，约 9 轮回本
- 进修消耗 SAN/HP：开始 -5 SAN，每轮 -4 SAN / -2 HP
- 完成时 SAN +10
- 进修期间挨饿 → 进修取消，进度清零
- SAN < 30 或 HP < 30 时不能开始进修

## 医疗灾难

| 参数 | 值 |
|------|-----|
| 发生时机 | 第 10-25 轮之间随机一轮，仅一次 |
| 医疗费 | $100/人（所有玩家同时） |
| 付得起 | 直接扣除，SAN -10 |
| 付不起 | 资产归零 → 贷款，SAN -25 |

## 灾难贷款

| 参数 | 值 |
|------|-----|
| 利率 | 20% |
| 还款期 | 20 轮 |
| 精神压力 | 每轮 SAN -2（直到还清） |
| 还清奖励 | SAN +10 |

## 打工疲劳

- 连续打工 1-2 轮：无惩罚
- 连续打工 ≥ 3 轮：每轮 SAN -3, HP -2
- 做任何非 WORK 行动（GAMBLE/STUDY/BUY_GOODS/BUY_MEDICINE/REST）重置计数

## 智能跳过

引擎自动跳过 LLM 调用的情况：
1. 三者皆死：无论 WORK、GAMBLE 赢、GAMBLE 输都活不了 → 强制打工
2. 进修中：锁定期间无需决策

## 设计理念

### 不是"谁钱最多谁赢"

传统的"资产最高者胜"让 SAN/HP 变成纯成本。幸福人生榜鼓励玩家在赚钱、健康、快乐之间权衡。一个始终稳定、偶尔休息、保持健康的玩家，可以击败拼命打工但精神崩溃的对手。

### SAN 和 HP 有明确分工

- HP 影响"能不能干活"：打工效率、疾病阈值、死亡风险
- SAN 影响"能不能理智决策"：赌博风险、进修能力、精神崩溃

两者不再互为镜像。

### 正面事件很重要

不只是惩罚——赌博赢、进修完成、还清贷款都带来 SAN 奖励。这让"追求快乐"成为合理策略，不只是"逃避痛苦"。

### 软惩罚优先于硬淘汰

SAN/HP 低时先降低效率（打工打折、进修变慢、小病概率上升），再威胁生命。给玩家抢救窗口，不是突然暴毙。

### REST 给穷人退路

免费的休息动作让没钱的人也能恢复，不用只能等死。效率低但有选择。
